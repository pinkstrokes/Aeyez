# MMMU Result Conversion Utilities

This package provides utilities for converting MMMU flow evaluation results between different formats.

## Overview

The `result_converter.py` module contains functions to convert individual question JSON files (generated by the MMMU flow) into:
1. **Summary CSV files** - Human-readable format with all question details and responses
2. **Evaluation JSONL files** - Format compatible with the existing MMMU evaluation pipeline

## Functions

### `generate_summary_csv_from_questions(session_dir, original_data, output_file, task)`
Generates a comprehensive CSV summary from question JSON files.

**Output CSV contains:**
- Question metadata (id, index, question text, options A-D)
- Dataset information (split, topic_difficulty, subfield, etc.)
- Model responses and expected answers
- Execution statistics (duration, token usage, message count)
- Image presence indicators

### `generate_evaluation_jsonl_from_questions(session_dir, original_data, output_file, task)`
Generates JSONL file compatible with existing MMMU evaluation tools.

**Output format:**
```json
{
  "question_id": 1,
  "annotation": {...},  // Original dataset row (cleaned)
  "task": "MMMU_DEV_VAL",
  "result": {"gen": "A"},  // Model response
  "flow_metadata": {...}  // Flow execution details
}
```

### `convert_session_results(session_dir, original_data_path, task, generate_csv, generate_jsonl)`
Convenience function to generate both CSV and JSONL outputs.

## Usage

### In Python Code
```python
from result_converter import convert_session_results
from pathlib import Path
import pandas as pd

# Convert a session directory
convert_session_results(
    session_dir=Path("logs/mmmu/session_20250921_141131_293"),
    original_data_path=Path("data/MMMU_DEV_VAL.tsv"),
    task="MMMU_DEV_VAL"
)
```

### Using run_mmmu_flow.py
The functions are automatically used by `run_mmmu_flow.py` when running inference:

```bash
python run_mmmu_flow.py infer --max-samples 10
# Automatically generates summary.csv and inference_results.jsonl

# Then evaluate the results
python run_mmmu_flow.py eval --input-file logs/mmmu/session_*/inference_results.jsonl
```

### Standalone Conversion Script
Use the root-level `convert_session_results.py` script to convert existing results:

```bash
# Convert existing session directory
python convert_session_results.py logs/mmmu/session_20250921_141131_293 data/MMMU_DEV_VAL.tsv

# Generate only CSV summary
python convert_session_results.py logs/mmmu/session_20250921_141131_293 data/MMMU_DEV_VAL.tsv --no-jsonl

# Custom task name
python convert_session_results.py logs/mmmu/session_20250921_141131_293 data/MMMU_DEV_VAL.tsv --task MY_TASK
```

## File Structure

```
session_directory/
├── question_1.json          # Individual question results
├── question_2.json
├── ...
├── summary.csv              # Generated CSV summary
├── inference_results.jsonl  # Generated JSONL for evaluation
└── session_info.txt         # Session metadata
```

## Requirements

- pandas
- pathlib
- json (built-in)

## Integration

These utilities are designed to:
1. **Work with existing MMMU evaluation pipeline** - JSONL output is compatible
2. **Provide human-readable summaries** - CSV format for analysis
3. **Support batch conversion** - Convert multiple session directories
4. **Maintain data integrity** - Preserve all original dataset fields and metadata
